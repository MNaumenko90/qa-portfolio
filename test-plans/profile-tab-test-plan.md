# Тест-план: Раздел «Профиль» мобильного приложения (Android)

> **Контекст кейса:** демонстрационный тест-план, составленный по мотивам реального опыта тестирования мобильного e-commerce/сервисного приложения. Структура и подход к декомпозиции — рабочие, названия экранов и логика адаптированы под гипотетическое приложение "AutoCare" (агрегатор услуг для автовладельцев), чтобы не раскрывать данные конкретного работодателя.

**Платформа:** Android
**Фича:** Обновлённый экран «Профиль» — гостевой режим, авторизованный пользователь, настройки

---

## 1. Гостевой режим (Guest Mode)

- [ ] Отображается текст-заглушка "Вы в гостевом режиме. Войдите, чтобы разблокировать все функции"
- [ ] Кнопка «Войти / Зарегистрироваться» — текст не обрезан, выравнивание корректно
- [ ] Кнопка входа через Google — работает авторизация
- [ ] Проверить отображение баннера с акциями в гостевом режиме
- [ ] Точка входа в «Настройки» скрыта/недоступна для гостя
- [ ] Проверить доступность приложения в гостевом режиме (нет краша, нет блокирующих экранов)

## 2. Шапка профиля — авторизованный пользователь

- [ ] Аватар отображается (или заглушка в шахматную клетку, если фото не задано)
- [ ] Полное имя и email отображаются корректно
- [ ] Числовой ID пользователя отображается под блоком профиля (не редактируется)
- [ ] Ссылка «Редактировать личные данные» кликабельна и открывает нужный экран

## 3. Раскладка и системная логика

- [ ] Порядок виджетов сверху вниз соответствует макету (Гараж → Заказы → Промо-баннер → Избранное → Бонусы → Адрес доставки → Поддержка)
- [ ] На экранах высотой 640px и ниже — элементы сжимаются пропорционально, заголовки и текст не перекрываются
- [ ] При загрузке отображается skeleton loader (серые плейсхолдеры) на всех виджетах
- [ ] При ошибке загрузки (таймаут >10с) — инлайн-сообщение "Не удалось загрузить [раздел]. Нажмите, чтобы повторить" — каждый виджет повторяет попытку независимо от других
- [ ] Остальные виджеты остаются доступны, если один из них не загрузился
- [ ] Страница скроллится плавно, нижняя навигация не перекрывает контент
- [ ] Счётчики на бейджах виджетов соответствуют реальному количеству элементов (машины, заказы, избранное)
- [ ] Отступы между блоками соответствуют макету
- [ ] Ссылка на политику конфиденциальности кликабельна

## 4. Настройки

- [ ] Поля «Имя» и «Email» заблокированы для редактирования (read-only) на главном экране настроек
- [ ] ID пользователя отображается как статичное поле
- [ ] Присутствуют все пункты навигации: реквизиты, подписки, депозитный счёт, смена пароля, текущая страна, системные настройки, политика конфиденциальности, оценить приложение

### Подэкран «Личные данные»
- [ ] Поля «Имя» и «Фамилия» редактируемые
- [ ] Поле Email — read-only
- [ ] Кнопка «Сохранить» (✓) в тулбаре активируется только после изменения хотя бы одного поля
- [ ] Нажатие на аватар открывает bottom sheet «Как вы хотите изменить фото профиля?» с опциями: выбрать из галереи / сделать новое фото / удалить фото / отмена
- [ ] Добавление фото из галереи работает
- [ ] Съёмка фото с камеры работает
- [ ] Загруженное фото обрезается в круг 56×56px без искажений/растяжения
- [ ] Загрузка файла не JPG/PNG или больше 5MB — выдаёт ошибку
- [ ] «Удалить фото» — сбрасывает аватар на дефолтную заглушку
- [ ] После возврата с экрана редактирования профиля — главный экран рендерится корректно (нет проблем с бордерами/лейаутом)

### Смена пароля
- [ ] Это отдельный полноценный экран (не встроенный в личные данные)
- [ ] Плейсхолдер поля пароля имеет корректные отступы
- [ ] Блок смены пароля отображается корректно для гостевых пользователей (заблокирован/скрыт)

### Текущая страна
- [ ] Текущая страна отображается с флагом
- [ ] Нажатие открывает экран управления адресами

## 5. Оценить приложение

- [ ] Выбор 4–5 звёзд → редирект в Google Play
- [ ] Выбор 1–3 звёзд → открывает почтовый клиент с предзаполненным адресом поддержки (если магазин приложений не открывается)
- [ ] Без выбора оценки — блок остаётся в исходном состоянии, действие не триггерится

## 6. Виджет «Гараж» (список автомобилей пользователя)

**Счётчик бейджа:**
- [ ] Бейдж в шапке = количеству сохранённых автомобилей, скрыт при 0

**Пустое состояние:**
- [ ] Отображается форма добавления авто инлайн
- [ ] Мотивационный заголовок "Добавьте автомобиль для точного подбора запчастей"

**Поиск по гос. номеру:**
- [ ] Флаг страны в поле поиска соответствует текущей стране пользователя (из Настроек → Текущая страна)
- [ ] Кнопка поиска активна во время ввода
- [ ] Успешный поиск → экран подтверждения с предзаполненной маркой/моделью
- [ ] Неудачный поиск → инлайн-сообщение "Автомобиль не найден. Введите данные вручную"

**Ручной ввод:**
- [ ] Три последовательных шага: марка → модель → модификация — каждый открывает полноэкранный селектор с поиском
- [ ] Каждый следующий шаг заблокирован, пока не выбран предыдущий
- [ ] Смена марки автоматически сбрасывает модель и модификацию
- [ ] Кнопка «Добавить в гараж» активна (не серая), только когда заполнены все три поля

**Обнаружение дублей:**
- [ ] Попытка добавить уже существующий автомобиль (тот же VIN или марка+модель+год+модификация) → инлайн "Этот автомобиль уже в вашем гараже", отправка блокируется

**Лимит карточек (1–20 автомобилей):**
- [ ] Карточки в один горизонтальный ряд с прокруткой
- [ ] При ровно 20 автомобилях кнопка «+ Добавить машину» пропадает из шапки виджета
- [ ] Если кнопка всё ещё видна и пользователь пытается добавить 21-ю — тултип "Вы достигли максимального количества автомобилей"

**Карточка автомобиля:**
- [ ] Фото автомобиля (кастомное, если задано, либо {Марка} {Модель} по умолчанию)
- [ ] Пробег с суффиксом "км"
- [ ] Дата последнего ТО
- [ ] Статус ТО (тег): красный (просрочено), жёлтый/янтарный (скоро), зелёный (всё ок)
- [ ] Нажатие на карточку открывает Детали автомобиля

## Примечания к подходу

- Чек-лист построен по принципу декомпозиции экрана на логические блоки (гостевой режим → авторизованный юзер → системная логика → под-фичи), а не по хронологии кликов — это ускоряет как написание, так и последующий регрессионный прогон, потому что легко видеть, какой блок независим от других.
- Отдельное внимание уделено граничным состояниям (0 элементов, лимит элементов, дубликаты, таймауты сети) — именно они чаще всего остаются непротестированными при поверхностном подходе.
- Пункты сгруппированы так, чтобы каждый мог тестироваться и репортиться независимо — это упрощает распараллеливание тестирования между несколькими QA.
