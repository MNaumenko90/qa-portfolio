# Тестирование Swag Labs (saucedemo.com)

saucedemo.com — открытый demo-магазин, которым обычно пользуются для тренировки тестировщиков. Ниже — тест-кейсы и баги, которые я нашла при прогоне логина, каталога и оформления заказа. Сайт публичный, так что все шаги можно повторить самому.

Тестовые аккаунты: standard_user, problem_user, performance_glitch_user, locked_out_user. Пароль везде secret_sauce.

## Тест-кейсы: авторизация

| ID | Название | Шаги | Ожидаемый результат |
|----|----------|------|----------------------|
| TC-01 | Успешный вход | Ввести standard_user / secret_sauce, нажать Login | Переход на страницу товаров |
| TC-02 | Заблокированный пользователь | Ввести locked_out_user / secret_sauce | Ошибка "Sorry, this user has been locked out" |
| TC-03 | Неверный пароль | Ввести standard_user с неверным паролем | Ошибка о неверных данных, поля не сбрасываются |
| TC-04 | Пустые поля | Оставить оба поля пустыми, нажать Login | Ошибка "Username is required" |
| TC-05 | Только Username | Заполнить только Username | Ошибка "Password is required" |
| TC-06 | Регистр в логине | Ввести Standard_User с заглавной буквы | Ожидается ошибка — логин регистрозависимый |
| TC-07 | Вход через Enter | Ввести данные, нажать Enter в поле пароля | Работает так же, как клик по кнопке |

## Баги на аккаунте problem_user

Этот аккаунт специально сделан "сломанным" — на нём удобно тренироваться находить баги.

1. Картинки товаров перепутаны — везде показывается одна и та же картинка вместо реальных товаров.
2. Сортировка по цене не работает: выбираешь "Price (low to high)", а порядок не меняется.
3. В поле Last Name на checkout нельзя ничего ввести — поле есть, но не реагирует на клавиатуру.

## Баг на performance_glitch_user

4. Каталог грузится заметно дольше (5+ секунд), чем у обычного пользователя. Похоже на искусственную задержку.

## Пример баг-репорта

Заголовок: [Web] Сортировка по цене не работает у problem_user

Приоритет: Medium
Компонент: Каталог / Sorting

Ожидаемое поведение: при выборе "Price (low to high)" товары должны выстроиться по возрастанию цены.

Фактическое поведение: список остаётся в исходном порядке (по имени), цена не учитывается.

Шаги воспроизведения:
1. Открыть saucedemo.com
2. Войти как problem_user / secret_sauce
3. На странице Products открыть сортировку
4. Выбрать "Price (low to high)"
5. Сравнить порядок цен до и после

Ожидаемый результат: 7.99 → 9.99 → 15.99 → 29.99 → 49.99

Фактический результат: порядок не меняется, хотя дропдаун показывает, что опция выбрана.

Окружение: Chrome, веб, пользователь problem_user.

Почему Medium, а не выше: сортировка не блокирует покупку, но заметно ломает заявленную функциональность.

На standard_user этот баг не воспроизводится — сортировка там работает нормально, значит дело не в сортировке вообще, а в конкретном тестовом сценарии.
